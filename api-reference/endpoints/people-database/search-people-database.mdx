This endpoint allows you to query our People database.

## Request body

- **filters** (required): Array of filter objects
	- **filterId**: Filter identifier (string)
	- **in**: List of values you want to include (array)
	- **out**: List of values you want to exclude (array)
- **page** (optional): Page number (number)
- **size** (optional): Size of the results, number between 1 and 100 (number)
- **excludes** (optional): Array of properties to exclude from the results (array)

### Filters

Each filter should have the following properties:
- **filterId**: Filter identifier (string)
- **in**: List of the values you want to include (array)
- **out**: List of the values you want to exclude (array)

To know which filters you are able to use, refer to the [GET Filters](/api-reference/endpoints/people-database/get-database-filters) section.

Example

```json
{
	"filters": [
		{
			"filterId": "country",
			"in": ["France"],
			"out": []
		},
        {
            "filterId": "currentCompany",
            "in": ["Lemlist"],
            "out": ["Lemwarm"]
        }
	],
	"page": 1,
	"size": 25,
	"excludes": ["education", "skills"]
}
```

## Response

Response will be formatted as an object with the following properties:
- **results**: Results array
- **total**: Total number of results (number)
- **took**: Duration of the elastic-search query in milliseconds (number)
- **page**: Current page (number)
- **size**: Size of the results (asked through parameters) (number)
- **search**: Unique identifier of the current search (string)
- **limitations**: Remaining calls for the POST People database and POST Companies database (search is limited to a number of queries every 24 hours) (number)
- **team**: Unique identifier of the current team (string)

If you need more details about the schema of the people returned as a results array, refer to the [GET People Schema](/api-reference/endpoints/people-database/get-people-schema) section.
